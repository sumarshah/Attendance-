# Free / Low-Cost Options (If You Don’t Want To Pay For VPS)

## Option A: Keep It Local (Office PC)
- Run backend + database on a dedicated office PC.
- Access via LAN or VPN.
- Cheapest, but less reliable than a VPS.

## Option B: Cloud Free Tiers (Limited)
- Some providers have free tiers, but databases or always-on services are usually limited.
- Good for demo/testing, not for production attendance.

## Option C: Shared Hosting
- Not ideal for Docker + Postgres + API.
- Works only if you rebuild to fit their stack.

## Recommendation
For attendance (business critical), VPS is the best long-term path.
